v1.0.0 前期调研
-前期调研已完成，调研文档更新

v1.0.1 手动prompt验证
- 先采用手动 prompt工程的方式实现简单的想法验证
- 设计了一个初步的系统提示词，可以让模型输出一个粗略的行程规划 JSON，包含每天的住宿地点、景点和交通方式等信息
- 初步测试了这个提示词，模型能够按照要求输出 JSON，内容基本合理


v1.0.2 多组件化
- 主要是从单点对话升级到多组件协作，用于验证规划的合理性，并且从代码设计的角度而非仅仅对话交互来构建一个标准的规划agent
- 建立一个Planner-Executor
- Planner (规划器)： 负责理解需求，输出你现在的这种“粗略 JSON”。
- Validator (校验器)： 这是一个纯代码逻辑模块，用来检查 AI 输出的 JSON。比如：检查两点间距离是否合理，时间是否重叠。
- Refiner (修正器)： 如果校验失败，将错误反馈给 AI，让它“重写”特定 Slot

v1.0.3 Function Calling
- 接入 Google Maps / Mapbox API： 编写代码提取 JSON 里的 geo_data，调用 API 获取真实的行驶里程和时间
- 接入天气或营业时间数据： 比如通过 API 发现 5 月 15 日某景点闭馆，Agent 必须自动调整行程

v1.0.4 RAG
- 构建一个小型知识库，存储用户的偏好、历史行程
- 通过向量检索让 Agent 能够动态获取这些信息，进行个性化推荐和调整
  

v1.0.5 评估
