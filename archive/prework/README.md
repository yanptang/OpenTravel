# Prework

这里保留项目早期的调研、需求整理和 prompt 设计记录。当前阶段不再把这些内容当作主线文档，而是作为背景材料和决策依据保留。

## 保留内容的目的

- 记录项目最初的设计思路
- 保留需求从粗到细的演进轨迹
- 方便后续回看为什么采用本地 CLI、slot JSON 和分段生成

## 文件索引

- `调研信息.md`：早期方案调研与技术选择记录
- `手动prompt.md`：最初的 prompt 设计和实验记录
- `实施计划.md`：阶段性推进计划
- `next_phase_plan.md`：下一阶段的开发重点与优先级
- `待整理信息.md`：尚未完全归纳的零散信息
- `开发日志.md`：早期开发记录

## 核心结论

前置工作的主要结论已经收敛到当前主线：

1. 先做本地 CLI 闭环，再考虑前端。
2. 行程输出必须结构化，才能做后端校验和局部修改。
3. 小模型更适合分段生成，而不是一次性生成整段长攻略。
4. 输入阶段就要补齐出发地、到达方式和 must-do 这类关键字段。
