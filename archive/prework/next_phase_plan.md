# Next Phase Plan

## 阶段目标

第二阶段的重点不是接入更多外部信息源，而是先把生成质量的基础能力做扎实。当前最优先的三个能力是：

1. 多轮澄清
2. 时间线式行程组织
3. 跨天闭环与交通衔接

偏好记忆放在后续阶段，不作为当前最优先任务。

## 能力优先级

### 1. 多轮澄清

目标是让系统像真实旅游管家一样，先把关键信息补齐，再进入正式生成。

建议优先追问的信息包括：

- 出发地
- 到达方式
- 出行日期
- 人数
- 预算范围
- 交通方式
- 住宿偏好
- 是否自驾
- 是否有必须安排的活动
- 是否想参加当地特色活动

澄清策略建议采用“少量、分批、可选项明确”的方式，而不是一次性抛出过多问题。

推荐做法：

- 先问必填项，保证行程可以生成
- 再问偏好项，提升行程质量
- 每轮只问 2 到 4 个问题
- 尽量给出选项，而不是开放式追问

### 2. 时间线式行程

调研里最稳定的结论是，旅行规划的核心不是景点列表，而是时间线和衔接关系。

后续生成结果应从“按天列表”升级为“按天时间线”，让每个 slot 都具备明确的起止时间、地点和衔接逻辑。

建议强化的内容：

- 上午、下午、晚上按时间段组织
- 每个活动前后都要有交通或缓冲 slot
- 每天的住宿点要自然衔接到第二天
- 行程尽量避免跳点和折返

### 3. 跨天闭环

跨天闭环是当前最值得继续补强的规则层能力。

建议至少保证：

- 到达日必须包含交通衔接
- 每天最后一个 slot 必须回到酒店
- 次日第一个 slot 要和前一晚住宿地形成合理连接
- 最后一天必须考虑返程交通

## 记忆规划

偏好记忆放在第二阶段之后再做。

原因是当前阶段更需要先把基础规划链路稳定下来，避免在“记住什么”还没定义清楚时就提前写入复杂记忆逻辑。

较合理的顺序是：

1. 先完成多轮澄清
2. 再完成时间线与跨天闭环
3. 然后引入偏好记忆
4. 最后再考虑更复杂的知识增强或外部工具接入

## 建议的交互方式

用户初次输入目的地后，系统不应直接生成完整攻略，而应先启动澄清流程。

例如：

- 用户输入“去冰岛，9 天，自驾”
- 系统先确认出发地、人数、到达方式、预算、住宿偏好
- 再主动询问是否要加入当地特色活动
- 最后根据偏好生成更完整的行程

这一点更像真实旅游管家，也更符合调研里高质量 agent 的行为模式。

## 当前结论

第二阶段的目标应该聚焦在“问得更准、排得更顺、跨天更稳”。

在这一阶段完成后，再把偏好记忆作为第三步引入，会更自然，也更不容易返工。
